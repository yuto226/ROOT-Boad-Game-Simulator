# Root シミュレーター開発ロードマップ

## 前提・進め方の方針

- **1フェーズ = 1〜2回のチャットセッションで完結する粒度**に分割する
- 各フェーズの最後に **「次のセッションへの引き継ぎメモ」** を必ず更新する
- コードは都度 `/mnt/user-data/outputs` 経由で受け取り、**ローカル(自分のPC)にリポジトリとして蓄積**していく前提
  - Claudeとの会話はセッションごとにリセットされるので、コード本体・設計判断・既知のバグは全部ファイル(このmdやコード内docstring)に残す
  - 新しいチャットを始めるときは「このロードマップmd」+「該当フェーズの既存コード」をアップロードして再開する運用にする
- 1セッションでやることが多すぎると判断したら、**その場で更に分割を提案する**(無理に一気にやらない)

## フェーズ6: 強化学習(RL)層の追加(2026-07-10 ユーザー構想確定により具体化)

**目的**: PPO self-playで学習したbotを載せる。学習はWindows機(RTX 4080 Super)のDocker環境で実行し、MacからSSHで制御する(PCを起動しておくだけの運用)。

**方針(Fable提言・ユーザー合意)**:
- 手法はまず **PPO+合法手マスク**(AlphaZero型MCTSは純Pythonエンジンでは計算量が合わない)
- 最初は **2人戦固定マッチアップ(例: 猫vs鷲巣)・完全情報(手札可視)** で割り切る。4人戦・隠匿情報は学習が回ってから
- エンジン本体は標準ライブラリのみを維持し、学習スタックとは依存分離(エンジンをpipインストール可能化してコンテナ側から `pip install -e`)
- 規模感の注意: 100万試合≈10⁸〜10⁹ステップ(AlphaZero級)。まず10⁷〜10⁸ステップ(数万〜数十万試合)から。エンジン速度が律速になったらホットパスのRust/C++移植を検討(その際も `legal_actions`/`apply` のインターフェースと決定性制約=DESIGN.md 10.2が仕様書になる)

### 6a: RL環境ラッパー(Macだけで開発・テスト可能、GPU不要)
- [ ] エンジンのpipインストール可能化(pyproject.toml)
- [ ] 観測エンコーダ(GameState→テンソル)
- [ ] 行動の固定インデックス化+合法手マスク(`legal_actions` から生成)
- [ ] PettingZoo AEC型環境ラッパー(pendingデシジョン=1ステップ1意思決定に写像)

### 6b: Windows側の学習環境構築
- [ ] WSL2+Docker+NVIDIA Container Toolkit(RTX 4080 SuperをCUDAコンテナから使う)
- [ ] WSL2内sshd自動起動+Tailscale(Macからポート開放なしでSSH到達)
- [ ] 学習ジョブはdetachedコンテナ(`docker run -d`)+チェックポイント定期保存。SSH切断で学習が死なない構成
- [ ] W&B or TensorBoardでMacのブラウザから学習曲線を監視
