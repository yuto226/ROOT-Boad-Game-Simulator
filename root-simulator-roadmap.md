# Root シミュレーター開発ロードマップ

## 前提・進め方の方針

- **1フェーズ = 1〜2回のチャットセッションで完結する粒度**に分割する
- 各フェーズの最後に **「次のセッションへの引き継ぎメモ」** を必ず更新する
- コードは都度 `/mnt/user-data/outputs` 経由で受け取り、**ローカル(自分のPC)にリポジトリとして蓄積**していく前提
  - Claudeとの会話はセッションごとにリセットされるので、コード本体・設計判断・既知のバグは全部ファイル(このmdやコード内docstring)に残す
  - 新しいチャットを始めるときは「このロードマップmd」+「該当フェーズの既存コード」をアップロードして再開する運用にする
- 1セッションでやることが多すぎると判断したら、**その場で更に分割を提案する**(無理に一気にやらない)

## 運用ルール(セッションをまたぐときのチェックリスト)
